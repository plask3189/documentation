Redesign of the UC Shuttle Site

Contributors: Dylan Bok, Casey Jackson, Andrew Pipo, Kate Plas

Description of Project

This project was tailored around redesigning the UC Shuttle Application/Website, currently utilized to track shuttle locations along their routes. Issues with the original application include difficulties in viewing information, bugged functionalities, and a cluttered interface. Our redesign focuses on a more user-friendly layout, ensuring relevant information is presentable and specific to selected routes.

Design Work

For our design, we focused on simplifying the map and adding an intuitive search feature. Interviews with users confirmed this approach. Below are the key questions and responses from our user interviews:

Interview Questions:

  • What do you expect when using an app or website for public transportation?
  • When using an app or website to track something in real-time, what is most important to you?
  • Would you like delays, early arrivals, and cancellations to be clearly visible in the interface?
  • Have you ever used the UC shuttle app or website?
  • When looking at this image of the shuttle tracker, is there anything unclear?
  • Is there anything missing or not easily accessible?
  • Is there anything you would remove?

User Responses:

Responses varied, but common themes included the need for accuracy, clear visibility of delays or cancellations, and less cluttered interfaces. Many users suggested removing unnecessary elements and prioritizing route-specific information.

Sketches

We created multiple sketches, including storyboard sketches, to conceptualize the redesign. Our final design includes:

  • A map on the right side with routes hidden by default to reduce clutter.
  • A left-side panel featuring shuttle information, a search area for starting and destination stops, and next bus arrival times.
  • Timetable functionality to dynamically update based on user input.

Background Research

Our research included examining the Transit Mobile App and airline apps for inspiration. We adopted features such as real-time updates and query-first interfaces, integrating these into our redesign.

Interface in Detail

The redesign includes features such as live maps using the Google Maps API, a search function, shuttle operation hours, and a timetable. Live shuttle locations and interactive stops enhance user experience.

Implementation

We used the Google Maps API for live map integration and the Svelte framework for the website. The project is hosted on Vercel.

AI Usage

AI tools were used to resolve issues with the Google Maps API and container layout, enhancing the overall design.

Future Work

Future enhancements include implementing additional routes, intelligent routing algorithms, and intuitive visual systems to distinguish routes.
